ISCX VPN-NonVPN Traffic Dataset 2016

A benchmark dataset released by the University of New Brunswick containing labelled network flows captured across 14 real-world traffic categories — both over standard connections and through VPN tunnels — enabling rigorous evaluation of encrypted traffic classifiers.

How this dataset was collected: Traffic was captured in a controlled lab environment at the University of New Brunswick using Wireshark on a mirrored switch port. Seven real applications were used — both directly and through a VPN tunnel — generating paired labelled flows for each scenario. CICFlowMeter was then applied to extract 80+ statistical features from the raw PCAP files.
14 Traffic Classes
Seven application types captured both with and without VPN tunnelling
🌐
Browsing Non-VPNVPN
HTTP/HTTPS web browsing using Firefox and Chrome on popular websites. Characterised by bursty short flows with mixed packet sizes.
🎬
Streaming Non-VPNVPN
Video streaming via Vimeo and YouTube. Long-lived flows with large, consistent downstream packet sizes and stable inter-arrival times.
💬
Chat Non-VPNVPN
Instant messaging over AIM, Facebook, Hangouts, ICQ, Skype. Short, frequent bidirectional flows with small payload sizes.
📞
VoIP Non-VPNVPN
Voice over IP calls via Facebook and Skype audio. Highly regular inter-arrival times, fixed small packet sizes, real-time bidirectional flow.
📁
File Transfer Non-VPNVPN
FTP, SFTP, Skype file transfers. Large sustained throughput with maximum-sized packets and low inter-arrival variance.
📧
Email Non-VPNVPN
Email via SMTPS, IMAPS using Gmail and Thunderbird. Asymmetric flows — large upstream during send, bursty during inbox sync.
🎮
P2P Non-VPNVPN
BitTorrent peer-to-peer file sharing. High fan-out connections, symmetric bidirectional traffic, variable packet sizes.

Dataset Characteristics
  • Traffic was captured in a controlled lab environment to ensure clean, unambiguous labels — each flow is associated with exactly one application and mode.
  • Both TCP and UDP protocols are represented, enabling the classifier to handle diverse transport-layer behaviour.
  • VPN traffic was routed through OpenVPN, a widely-used tunnelling protocol, making the encrypted flows genuinely challenging to classify.
  • The dataset exhibits natural class imbalance — VoIP and streaming classes have more flows than email or P2P — requiring careful evaluation with per-class metrics.
  • Features were extracted using CICFlowMeter, the same tool used in production — ensuring zero distribution shift between training and inference.
  • The dataset is publicly available from the Canadian Institute for Cybersecurity (UNB) and is a widely cited benchmark in network traffic classification research.
