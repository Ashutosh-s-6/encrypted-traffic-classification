Encrypted Traffic Classification Using Machine Learning

A flow-based machine learning system that classifies encrypted network traffic without payload inspection — preserving privacy while enabling deep network visibility using statistical flow features extracted from PCAP captures.

Problem Statement
  • Traditional DPI (Deep Packet Inspection) cannot classify encrypted traffic such as TLS/HTTPS — payload is hidden by design.
  • VPN traffic obscures application identity, making conventional signature-based classification ineffective.
  • Network administrators need visibility into traffic types for QoS, security policy enforcement, and anomaly detection.
  • A privacy-preserving classification method is needed that works on flow-level metadata rather than packet payloads.
Our Approach
  • Extract statistical flow features (inter-packet timing, packet lengths, byte ratios) from raw network captures using CICFlowMeter.
  • Train ensemble classifiers (Random Forest, XGBoost) on labelled ISCX-VPN-2016 flows — no payload required.
  • Classify uploaded CSVs in real time via a Flask web dashboard, displaying distribution, metrics, and per-class breakdown.
  • Provide an AI assistant for result interpretation, enabling non-expert users to understand the output.
Project Objectives
Core goals this system is designed to achieve
OBJ — 01

Flow-Level Feature Extraction

Extract 80+ statistical features from PCAP captures — packet length distributions, IAT stats, flag counts, flow duration — without accessing payload content.

OBJ — 02

Multi-Class Traffic Classification

Distinguish 14 traffic categories including VPN and non-VPN variants of streaming, browsing, chat, VOIP, file transfer, and email.

OBJ — 03

Model Benchmarking

Compare Random Forest and XGBoost on accuracy, precision, recall, and F1 — identifying the best classifier for encrypted traffic scenarios.

OBJ — 04

Real-Time Web Classification

Deploy a Flask dashboard that accepts any CICFlowMeter-generated CSV and returns full classification results within seconds.

OBJ — 05

Interactive Result Visualisation

Present traffic distribution, per-class breakdowns, and performance metrics through an interactive, enterprise-grade dashboard UI.

OBJ — 06

Accessible AI Interpretation

Embed a context-aware AI assistant that explains classification results in plain language, making the system accessible to non-expert network users.

🌲 Random Forest Robust
 Bagging Ensemble Handles noisy features well, resistant to overfitting, fast inference, strong out-of-the-box performance on tabular data Large memory footprint at scale, less accurate on imbalanced classes without tuning General-purpose classification, interpretability via feature importance
⚡ XGBoost Fast
 Boosting Ensemble Excellent on imbalanced datasets, built-in regularisation, very high accuracy with tuning, handles missing values natively More hyperparameters to tune, slower to train than Random Forest Competitive accuracy benchmarking, imbalanced traffic class distributions
🌿 LightGBM Efficient
Gradient Boosting Extremely fast training, handles large datasets efficiently, better memory optimization, high accuracy in structured data Can overfit on small datasets, sensitive to noisy features, less interpretable compared to Random Forest Large-scale data, real-time systems, production environments
🧠 Key Insights
  • LightGBM generally provides the best performance due to efficient leaf-wise tree growth.
  • Random Forest is more stable and resistant to noisy features but may miss complex patterns.
  • XGBoost performs well on imbalanced datasets due to its boosting strategy.
  • F1 Score is the most important metric in this system as it balances precision and recall.
